git学习part2

2017-07-09 git

分支管理

注：一项工程的不同部分可以看作不同的分支

创建与合并

  • git checkout -b new-branch 创建并切换到新分支，-b参数表示切换到新分支

    上述命令等于以下两条命令

  1. git branch new-branch

  2. git checkout new-branch

  • git branch 查看分支目录，带*号为当前分支

  • git merge branch-name 把指定分支合并到当前分支

  • git branch -d branch-name 删除分支

    合并后可以删除该分支

  • git merge --no-ff -m "message" branch-name 以no-ff模式合并(no fast-forward),合并后branch不会被删除

bug & feature

  • git stash 临时储存当前工作区文件，把工作区留给临时任务

  • git stash list 查看临时储存区目录

  • git stash pop 弹出stash中的内容

    上述命令等于以下两条命令

    1. git stash apply
    2. git stash drop

  • git stash apply stash@{0} 恢复指定内容

  • git branch -D branch-name 删除未合并的分支

git学习part1

2017-07-06 git

初始化

用户设置

  • $ git config --global user.name "Your Name" 设置用户名

  • $ git config --global user.email "email@example.com" 设置用户邮箱

注：–global是全局参数，用了这个参数，表示你这台机器上所有的Git仓库都会使用这个配置，当然也可以对某个仓库指定不同的用户名和Email地址。

创建本地版本库

  1. mkdir folder-name 创建本地版本库文件夹
  2. cd folder-name 进入版本库
  3. pwd 显示当前目录，避免出错
  4. git init 初始化版本库
    把相应的文件放到版本库中就好

添加与提交

  • git add file-name(带上格式名) 把文件添加到仓库

git add .表示添加新文件和编辑过的文件, git add -A表示添加所有内容

  • git commit -m "提交说明" 把所有add的文件提交到仓库

版本控制

  • git log 显示提交日志

    即提交者，提交时间，提交说明，添加–pretty=oneline参数可以简化显示方式

  • git reset --hard HEAD^ 退回上一版本

    HEAD表示当前版本，HEAD^表示上一版本，也可以用git log 中的版本号前几位表示对应的版本

  • git reflog 查看历史命令

git工作模型

  • 工作区(Working Directory)

    工作区就是我们用mkdir命令创建的文件夹，我们对文件的修改就在这里进行

  • 版本库(Repository)

版本库就是.git文件夹，它分为暂存区(stage/index)和master分支,
git add 把文件添加到暂存区,
git commit 把文件添加到版本库

  • git status 查看仓库状态

撤销与删除

  • git checkout --file-name 舍弃修改，将工作区中该文件退回到上一次git add 或者 git commit的状态

  • rm file-name 删除工作区文件

远程仓库

推送

注：需要github账号

  1. ssh-keygen -t rsa -C "用户邮箱"
    创建ssh-key，确保安全性

>使用命令的默认值即可，由于这个Key也不是用于军事目的，所以也无需设置密码。如果一切顺利的话，可以在用户主目录里找到.ssh目录，里面有id_rsa和id_rsa.pub两个文件，这两个就是SSHKey的秘钥对，id_rsa是私钥，不能泄露出去，id_rsa.pub是公钥，可以放心地告诉任何人，然后把id_rsa.pub里的密钥添加到github中(廖雪峰的git教程)

  1. $ git remote add origin git@你的giyhub仓库地址 把本地库与GitHub上仓库关联

  2. $ git push origin master 把本地库推送到github上

第一次使用push命令要 把id_rsa.pub里的密钥添加到github中

克隆

git clone git@你的giyhub仓库地址
